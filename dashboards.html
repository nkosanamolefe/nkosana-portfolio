MySQL Power BI Streamlit

Goal

This project aims at creating an ETL pipeline that builds analytical tables for US air transport analysts with a focus on US domestic air travel. The tables will allow the analysts to investigate the links between US domestic flights, aircraft, engine types, airports, airlines and other variables so they can produce meaningful insights and MI.

Process

Data is downloaded onto local storage first, then moved into the cloud, to AWS S3 buckets, and then into Amazon Redshift. Once the Redshift cluster is up and running, the data is loaded from S3 to staging tables, and then from the staging tables to the final flights fact table and the date, time, aircraft, airlines, and airports dimension tables.

Highlights

The entire project is carried out using Python AWS SDK to automate as much of the ETL process as possible. The file upload to S3 buckets, creation of IAM role, and launch of Redshift clusters all happen programmatically – rather than using the AWS user interface in the browser, making the pipeline less prone to errors and more repeatable.
